Analyse applicative & journal technique

Toutes les métadonnées enregistrées pour comprendre l’usage de la plateforme, les frictions (connexions, recherches vides) et le comportement côté API. Aucun nom complet ni e-mail n’est affiché — uniquement des identifiants techniques, des libellés de routes ou des compteurs agrégés.

Types d’événements suivis (côté serveur)

  • Échec de connexion : tentative refusée (identifiant inconnu, compte désactivé ou mot de passe incorrect).
  • Recherche sans résultat : requête GET /v1/trips/search avec critères de villes / date et zéro trajet retourné (utile pour ajuster l’offre ou les textes d’aide).
  • Réservation créée / paiement confirmé : cycle de vente (création, puis validation portefeuille ou FedaPay).
  • Trajet publié : nouvelle offre enregistrée par un partenaire.
  • Erreur serveur : exception non gérée (classe d’exception enregistrée ; pas de fuite de données clients).
